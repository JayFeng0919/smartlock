# STM32F103 智能门锁

本工程基于 STM32F103C8T6 和 Keil MDK 5，实现密码、IC 卡、指纹和
OpenMV 二维码四种认证方式。

## 功能组成

- 4×4 矩阵键盘：密码输入和管理员菜单。
- RC522：IC 卡识别。
- AS608：指纹识别、录入和清空。
- OpenMV Cam H7 R2：二维码识别。
- OLED：状态和管理员菜单显示。
- PC13 LED、PB5 蜂鸣器：认证结果提示。

目前工程使用 LED 模拟门锁执行机构。以后增加继电器、舵机或电机时，
只需要在 `Hardware/Lock.c` 的统一开锁接口中添加控制代码。

## 引脚分配

| 模块 | STM32 引脚 |
|---|---|
| OpenMV I2C | PB6=SCL，PB7=SDA |
| OLED 软件 I2C | PB8=SCL，PB9=SDA |
| RC522 | PB1=RST，PA6=MISO，PA7=MOSI，PB10=SCK，PB11=NSS |
| 矩阵键盘行 | PA8、PA9、PA10、PA11 |
| 矩阵键盘列 | PB12、PB13、PB14、PB15 |
| AS608 串口 | PA2=TX，PA3=RX |
| AS608 其他引脚 | PA4=电源控制，PA0=TouchOut |
| 状态 LED | PC13，低电平点亮 |
| 蜂鸣器 | PB5 |

OpenMV 与 STM32 的接线：

| OpenMV Cam H7 R2 | STM32F103C8T6 |
|---|---|
| P7 / I2C4 SCL | PB6 / I2C1 SCL |
| P8 / I2C4 SDA | PB7 / I2C1 SDA |
| GND | GND |

SCL 和 SDA 分别通过约 4.7 kΩ 电阻上拉到 3.3 V，两块板必须共地。

## 二维码通信设计

OpenMV 始终作为 I2C 主机，STM32 始终作为地址 `0x12` 的 I2C1 从机。
STM32不需要主动向 OpenMV 发送命令。

OpenMV只负责识别并发送二维码原文，STM32负责保存和判断授权内容。

通信帧格式：

```text
A5 版本 类型 序号 长度 数据... CRC16低字节 CRC16高字节
```

- 协议版本：`0x01`
- 类型 `0x01`：识别到二维码，后面携带原文
- 类型 `0x02`：二维码已离开画面
- 二维码原文最大长度：64字节
- 校验算法：CRC16-CCITT，初始值 `0xFFFF`

同一个二维码停留在画面中时只会触发一次认证。二维码离开画面后，
才能再次触发。

配套的 OpenMV 程序位于工作区根目录：

```text
../untitled_2.py
```

调试完成后，应将它保存到 OpenMV 内部存储并命名为 `main.py`。

## 正常使用

### 密码开锁

按任意数字键开始输入四位密码。默认密码为 `0000`。

### IC 卡开锁

将卡片靠近 RC522。卡号在管理员模式中添加和删除。

### 指纹开锁

手指触发 AS608 的 TouchOut 后，系统自动进行指纹比对。

### 二维码开锁

把已经在管理员模式中录入的二维码放到 OpenMV 画面中。OpenMV发送
二维码原文，STM32比对成功后调用统一开锁接口。

## 管理员模式

长按 `#` 约1.5秒，输入管理员密码后进入管理模式。

一级菜单：

```text
1.Card  2.Finger
3.PW 4.QR  *.Exit
```

二维码菜单：

```text
1.Enroll 2.Clear
3.Status *.Back
```

### 录入二维码

1. 进入 `4.QR`。
2. 选择 `1.Enroll`。
3. OLED 显示 `Scan New QR` 后，把新二维码放到 OpenMV 前。
4. STM32显示收到的数据长度和部分内容。
5. 按 `#` 保存，按 `*` 取消。
6. 保存完成后先移开二维码，再退出管理员模式。

### 清除二维码

选择 `2.Clear`，再按 `#` 确认。清除后所有二维码都不能开锁，直到重新
录入。

### 查看状态

选择 `3.Status`，可以查看是否已经配置二维码以及原文长度。

## 二维码掉电保存

授权二维码保存在 STM32 内部 Flash 的最后两个 1 KB 页面：

```text
0x0800F800
0x0800FC00
```

两个页面采用 A/B 交替写入，记录包含版本号、序号和 CRC32。写入时最后
提交有效标记，降低保存过程中断电造成配置损坏的风险。

Keil 工程的程序 Flash 范围已经限制为：

```text
0x08000000～0x0800F7FF
```

不要把 IROM 大小重新改成 `0x10000`，否则程序可能覆盖二维码配置区。

重新下载整个芯片或执行全片擦除会清除已录入的二维码，这是正常现象。

## 编译与下载

1. 使用 Keil MDK 5 打开 `Project.uvprojx`。
2. 芯片选择 STM32F103C8。
3. 使用 ARM Compiler 5。
4. 按 `F7` 编译。
5. 在 Debug 中选择 ST-Link 和 SWD 接口。
6. 按 `F8` 下载。

## 目前需要注意

- 管理员密码和 IC 卡白名单仍然只保存在 RAM，断电后会恢复默认状态。
- 指纹模板保存在 AS608 自身的 Flash 中。
- 静态二维码可以被拍照复制，更高安全等级应使用动态二维码或二次认证。
